Docker 概述

Docker 是一个轻量化的虚拟化引擎。

Docker Engine

Docker Engine 是一个Client - Server应用程序，包含以下主要组件：

  • Server：即 Docker deamon，核心进程dockerd
  • REST API：与Docker deamon进行通讯的接口。
  • Client：即 Docker CLI ，一个命令行界面（CLI）客户端docker command

CLI 使用 Docker REST API 通过脚本或者直接CLI命令与Docker deamon进行交互。

Docker architecture

Docker 采用 Client - Server 架构。

Client 和 Deamon 可以在同一个系统上运行，也可以将 Client 连接到远端的 Docker deamon。

Docker deamon

Docker deamon（dockerd）侦听来自Docker API的请求，并管理 image、Container、Network 和 Volume 等对象。

守护进程还可以与其他守护进程通信，以管理docker服务。

Docker Client

Docker Client（docker）是与 Docker deamon最常用的主要交互方式。

例如：使用 docker run 命令，客户端会将这些命令发送给dockerd，然后由dockerd执行命令。

Docker registries

Docker registries 是一个存储 Docker image 的仓库。

例如：使用 docker pull 或 docker run 命令时，将从配置的 registries 中 提取所需的 image。

注：默认 registries 为 Docker Hub。

Docker 底层采用的基础技术

Docker 采用 Go语言编写，它利用Linux内核的几个功能来为其提供功能。

容器的底层实现技术：cgroup 和 namespace 是最重要的两种技术。cgroup 实现资源限额，namespace 实现资源隔离。

Namespaces

Docker 采用 Namespaces 技术提供的隔离空间被称为容器。

Namespaces 管理着 Linux OS 中全局唯一的资源，并且可以让每一个容器认为只有自己在使用这些资源。即 Namespace 实现了容器间的资源隔离。

运行容器时，Docker 会为该容器创建一组 Namespaces。 Namespaces 提供了隔离效果，容器的都在一个独立的 Namespaces 中运行，其访问权限仅限于该 Namespaces。

Docker Engine 使用了如下6种 Namespace：

  • Mnt namespace：管理文件系统挂载点，使容器看上去拥有整个文件系统，即容器拥有自己的 / 并且可以执行mount 和 umount。
  • UTS namespace：隔离内核和版本标识符，使容器拥有自己的hostname。默认下容器hostname是它的短ID，也可以通过-h 或 –hostname 参数设置。
  • IPC namespace：管理访问IPC资源，使容器拥有自己的共享内存和信号量（semaphore）来实现进程间的通信，而不会与host和其他容器的IPC混一起。
  • PID namespace：进程隔离，实现容器独立的一套PID，即 PID = 1 的进程不是 host 的 init 进程。
  • Net namespace：管理网络接口，使容器拥有自己独立的 网卡、IP、路由等网络资源。
  • User namespace：使容器能够管理自己的用户，host 无法查看到容器中创建的用户。

Control groups

cgroup 全称 Control Group 。Linux OS通过 cgroup 将进程限制为指定资源集，对进程使用CPU、内存、IO资源 进行约束。cgroup 允许 Docker Engine 将可用的硬件资源共享给容器，并对其可用资源进行约束和限制。

docker 资源限制中的 –cpu-shares 、-m 等参数，实际上是在配置 cgroup /sys/fs/cgroup/

Docker cgroup配置目录：

  • /sys/fs/cgroup/cpu/docker
  • /sys/fs/cgroup/memory/docker
  • /sys/fs/cgroup/blkio/docker

例：在 /sys/fs/cgroup/cpu/docker 目录中，Linux OS会为每个容器创建一个cgroup目录，以容器长ID命名。
